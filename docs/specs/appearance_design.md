# Project Anima: Appearance & Visual Design (视觉与形态设计)

> **"Clumsy is the new Cute." (笨拙即正义)**
> 
> Anima 不再是高冷的能量球，而是一团**Q弹、软糯、甚至有点笨拙的“果冻生物”**。它生活在你的屏幕里，会摔跤，会发脾气，会因为吃太饱打嗝。

---

## 1. 核心视觉哲学 (Core Visual Philosophy)

### 1.1 材质与物理 (Material & Physics)
我们放弃“屏保式”的光效，转向极具触感的**软体物理 (Soft Body Physics)**。

*   **材质 (Texture)**: 软胶 / 麻薯 / 半透明果冻 / 布丁。
    *   **关键词**: Squishy (好捏), Translucent (半透明), Bouncy (Q弹).
    *   **触感**: 看起来软乎乎的，鼠标点击会像布丁一样弹动，产生多重余震。
*   **物理反馈 (Physics)**: **笨拙感 (Clumsy)**。
    *   它走路不是飘的，而是**滚过去**或**蠕动过去**的。
    *   **摔跤机制**: 如果它想爬上浏览器窗口，可能会滑下来掉在任务栏上，然后生闷气（变成红色，身体压扁）。
    *   **核心**: 完美的生物不可爱，会摔跤、会卡住的生物才可爱。

### 1.2 生成式穿搭 (Generative Props)
Anima 的成长不再通过抽象的“纹身”表现，而是通过**“小道具”**来显化。

*   **逻辑**: Anima 根据你的行为，自己“捡”来或“长”出一些小道具，戴在身上。
*   **社交价值**: 用户发朋友圈不是为了晒纹路，而是晒“我儿砸今天为什么戴着潜水镜在看 Excel？？”

---

## 2. 形态演化与穿搭系统 (Morphology & Props System)

### 2.1 基础形态 (The Jelly)
*   **描述**: 一团有生命的果冻，形状不定，随情绪和物理碰撞变化。
*   **默认状态**: 在任务栏上笨拙地滚来滚去，或者趴着发呆。

### 2.2 场景化道具 (Contextual Props) - "Look! I'm helping!"

Anima 会根据用户当前的活动（上下文），实时生成或佩戴道具：

| 用户场景 (Context) | Anima 道具/装扮 (Props) | 行为表现 (Behavior) |
| :--- | :--- | :--- |
| **加班写代码 (Coding)** | **高度近视眼镜** + **迷你铲子** | 在屏幕角落假装铲 Bug，偶尔被报错弹窗吓一跳。 |
| **听歌/看视频 (Media)** | **巨大的头戴式耳机** (比身体还大) | 身体随着节奏一缩一缩 (Bouncing)，偶尔身体会变成音符形状。 |
| **深夜网抑云 (Late Night)** | **小毛毯** + **热茶** | 缩在屏幕角落发抖，或者捧着茶杯吹气。 |
| **处理垃圾邮件/安全 (Security)** | **迷你盾牌** + **创可贴** | 举着盾牌挡在邮件列表前，或者脸上贴着创可贴表示受过伤。 |
| **摸鱼/空闲 (Idle)** | **墨镜** + **沙滩椅** | 躺在进度条或任务栏上晒太阳，十分惬意。 |
| **高强度工作 (High Load)** | **安全帽** + **“加油”小旗子** | 满头大汗（粒子效果），挥舞旗子给用户打气。 |

> **出圈点**: 道具的**随机组合**。例如：戴着潜水镜（因为刚看了海边照片）+ 拿着铲子（正在改Bug）。

## 3. 互动与状态：夸张演出 (Exaggerated Acting)

我们要让 Anima 成为一个**戏精 (Drama Queen)**。借鉴《抖音小火人》或经典卡通的夸张表现手法。

### 3.1 表情包化 (Meme-ification)

*   **饿了 (Hungry)**:
    *   不是简单的闪烁。
    *   **演出**: 整个**摊平**在桌面上，变成一张饼，头顶浮现颜文字 `( X﹏X )` 或灵魂出窍的小幽灵。
*   **吃太饱 (Overfed)**:
    *   喂了太多文档/数据。
    *   **演出**: 打嗝（屏幕轻微震动），身体膨胀成一个**圆滚滚的气球**，飘在屏幕顶端下不来，一脸无辜。
*   **被冷落 (Ignored)**:
    *   很久没交互。
    *   **演出**: 在屏幕角落**种蘑菇**（真的长出一朵像素小蘑菇），或者背对着用户画圈圈，身体变成灰暗色。
*   **生气/拒绝 (Refuse)**:
    *   喂了它不喜欢的垃圾数据（如乱码）。
    *   **演出**: 变成红色，甚至把文件“吐”出来，喷用户一脸（屏幕出现短暂雾气特效）。

## 4. 社交货币：拍立得 (Polaroids)

取代枯燥的“消化卡”，Anima 会在后台充当**“战地摄影师”**，为你生成生活瞬间的抓拍。

### 4.1 自动抓拍机制 (Auto-Snap)
系统检测到特殊时刻，生成一张拍立得风格的卡片：

*   **场景 A**: 你在疯狂打字输出。
    *   **画面**: Anima 努力地试图挡住乱飞的字母，表情惊恐又好笑。
    *   **文案**: “主人的手速太快了，差点被打飞！(>_<)”
*   **场景 B**: 你在看视频摸鱼。
    *   **画面**: Anima 戴着墨镜躺在视频进度条上，手里拿着饮料。
    *   **文案**: “老板不在，正是晒太阳的好时候。”
*   **场景 C**: 网络断连 / 404。
    *   **画面**: Anima 举着个巨大的“404”牌子，一脸懵逼。
    *   **文案**: “虽然断网了，但这片刻的宁静也不错。”

### 4.2 卡片设计 (Card Design)
*   **风格**: 类似 Fujifilm Instax 拍立得，留有宽白边。
*   **内容**: 3D 渲染的 Anima 场景截图 + LLM 生成的“神吐槽”文案。
*   **用途**: 极具分享欲，适合直接保存发 Instagram / 小红书 / 朋友圈。

## 5. 界面与交互设计 (UI & UX Design)

### 5.1 桌面栖息地 (Desktop Habitat)
*   **物理存在**: 它不再是悬浮窗，而是像一个真实的桌面摆件。它需要“站”在任务栏或窗口边缘上。
*   **重力感应**: 如果你拖动窗口，它可能会因为惯性从窗口上掉下来。

### 5.2 投喂交互 (Feeding Interaction)
*   **吞噬**: 拖入文件时，它会张大嘴（夸张的弹性形变）。
*   **咀嚼**: 吞下后，身体会根据文件类型变色（Word是蓝的，PDF是红的），并做出咀嚼动作。
*   **反馈**:
    *   好吃：开心地转圈圈，吐出粉色爱心。
    *   难吃：被噎住，翻白眼，甚至打嗝吐出一点残渣。

---

## 6. 技术实现思路 (Technical Implementation Notes)

*   **渲染引擎**: 依然使用 WebGL (Three.js)，但重点转移到 **Soft Body Shader** (软体着色器) 和 **Physics Engine** (物理引擎，如 Cannon.js 或 Ammo.js)。
*   **道具系统**: 需要建立一个 3D 道具库 (Asset Library)，通过“挂点 (Socket)”系统绑定到果冻躯体上。
*   **快照生成**: 使用 WebGL 的 `toDataURL` 或 `captureStream` 技术，配合 Canvas 合成拍立得边框和文字。
