# 项目结构重构计划 (Project Restructuring Plan)

为了提升项目的可维护性与国际化标准，我计划对当前混乱的文件结构进行统一整理。

## 1. 目录结构重构 (Directory Structure)

我们将采用标准的 Monorepo 风格结构，将中文目录重命名为英文，并按功能分类：

```text
/
├── apps/                        # 核心应用程序代码
│   └── anima-web/               # (原 anima-web) 主线 Web/Tauri 前端
│
├── assets/                      # 静态资源
│   └── concepts/                # (原 概念图) 包含生成的设计图与演示视频
│
├── docs/                        # 项目文档中心 (原 说明文档 + 散落文件)
│   ├── business/                # 商业与市场文档
│   │   ├── startup_whitepaper.md   # (重命名) 创业白皮书
│   │   └── market_analysis.md      # (重命名) 市场分析报告
│   │   
│   ├── planning/                # 规划与进度
│   │   ├── technical_roadmap.md    # 技术路线图
│   │   └── project_progress.md     # (重命名) 项目进度记录
│   │   
│   └── specs/                   # 详细设计规范
│       ├── system_architecture.md  # (归档) 系统架构
│       ├── design_spec.md          # (原 design_project_anima.md)
│       ├── game_mechanics.md       # 游戏机制
│       └── appearance_design.md    # 视觉设计
│
├── prototypes/                  # 技术验证原型
│   ├── demo-01-jelly/           # (原 demo1) 果冻效果原型
│   ├── demo-02-react/           # (原 demo2) 组件化尝试原型
│   └── concept_proof.html       # 早期概念验证页面
│
├── .ai_status.md                # AI 记忆文件 (保留在根目录)
└── README.md                    # 项目入口文档 (更新索引)
```

## 2. 执行步骤 (Execution Steps)

1.  **创建目录**: 建立 `apps`, `assets/concepts`, `docs/{business,planning,specs}`, `prototypes`。
2.  **移动与重命名**: 使用 `git mv` 移动文件并重命名，保留 Git 提交历史。
3.  **清理**: 删除空的旧目录（`说明文档`, `概念图`, `demo1`, `demo2`）。
4.  **更新文档**: 重写根目录 `README.md`，更新所有文档的链接索引。
5.  **提交推送**: 将变更提交至 GitHub。

## 3. 命名规范 (Naming Convention)

*   **目录**: 全小写，使用连字符 (`kebab-case`)，如 `anima-web`。
*   **文档**: 全小写，使用下划线 (`snake_case`)，如 `game_mechanics.md`。
